Requerimientos-Funcionales
Súper usuario:
	R1.1: El sistema requiere tener un súper usuario el cual debe tener las características de:
		•	Único (solo 1 existente)
		•	Correo
		•	Contraseña (debe tener 6-10 debe contener al menos 1 mayúscula, 1 minúscula, 1 número)
			Y tener permisos hacia los perfiles de administrador para:
		•	Consultar (listar, ver sus datos)
		•	Modificar
			o	Nombres
			o	Apellidos
		•	Desactivar
	R1.2: El súper usuario podrá visualizar la lista de solicitudes pendientes de activación de usuario administrador donde podrá aceptar o rechazar.
	R1.3: El súper usuario tendrá forma de recuperar su contraseña por medio de un enlace, lo cual será un sistema que le solicita DNI recibirá al buzón de su correo un vínculo para realizar un cambio de contraseña	
	R1.4: El súper usuario puede modificar sus datos de 
		•	Contraseña
		•	Correo
	Para realizar estos cambios debe indicar su contraseña actual
	R1.5: El súper usuario puede crear, modificar, eliminar intereses (estos se usan para los usuarios egresados y la creación de noticias, eventos, actividad que crean administradores)
	
Administradores:
	R2.1: Los usuarios administradores tienen permisos en los usuarios egresados para:
		•	Modificar
			o	Nombres
			o	Apellidos
			o	Correo
		•	Visualizar
		•	Desactivar
		•	Activar
	R2.2: El usuario administrador en el módulo de noticias posee la facultad de: 
		•	Agregar (al agregar debe indicar los intereses relacionados, al menos 1)
		•	Modificar y agregar contenido de: 
			o	Audio
			o	Imágenes
			o	Videos u otros archivos (El contenido multimedia subido por el administrador se mostrarán en una sección de noticias o actividades para los usuarios egresados y administradores)
	R2.3: El administrador a sus datos puede de:
	solo Visualizarlos:
		•	DNI
		•	Nombres
		•	Apellidos
	Visualizar y Editar
		•	Departamento (donde vive)
		•	Ciudad (una vez que se elija el departamento de forma automática debe listar las ciudades correspondientes a este, cargado de forma automática, donde vive)
		•	Dirección
		•	Teléfono
		•	Correo - El correo es validado (formato correcto) antes de su creación
		•	Contraseña (esta no se puede visualizar, debe tener 6-10 debe contener al menos 1 mayúscula, 1 minúscula, 1 número)
	R2.4: Cuando un usuario egresado solicite inscribirse, el sistema debe confirmar los datos consultando en la base de datos de la información subida por el usuario y se le notificará a los administradores de su pendiente activación la cual podrá visualizar la lista de solicitud donde podrá aceptar o rechazar de los usuarios egresados
	R2.5: Para inscribirse en el sistema como usuario administrador primero debe de hacer una solicitud con los siguientes datos:
		•	Nombres
		•	Apellidos
		•	DNI
		•	Correo
	Esta solicitud queda pendiente para que la cuenta sea aceptada o rechazada, la cual puede ser vista y tratada por el súper usuario
	R2.6: Una vez que un usuario administrador que se ha inscrito en el sistema y su solicitud es aceptado por el súper usuario será Activado y recibirá un correo con una notificación y una contraseña generada
	R2.7: Cuando el usuario administrador ingresa por primera vez se le solicitará diligenciar la información faltante con respecto a lo que indica el requerimiento 2.3 lo cual sería:
		•	Departamento (donde vive)
		•	Ciudad (una vez que se elija el departamento de forma automática debe listar las ciudades correspondientes a este, cargado de forma automática, donde vive)
		•	Dirección
		•	Teléfono
		•	Ciudad
		•	Contraseña (una nueva, debe tener 6-10 debe contener al menos 1 mayúscula, 1 minúscula, 1 número)
	Tiene que diligenciar todos los datos para poder continuar, si no los completa cada nuevo login le solicitará ello.
		
Usuarios Egresados:
	R3.1: Para inscribirse en el sistema como usuario egresado primero debe de hacer una solicitud con los siguientes datos:
		•	DNI
		•	Nombres
		•	Apellidos
		•	Programa (donde egreso, dados de una lista)
		•	Correo
	Esta solicitud queda pendiente para que la cuenta sea aceptada o rechazada por un usuario administrador, solo permitirá enviar la inscripción si su DNI coincide con el programa que ha indicado
	R3.2: Una vez que un usuario egresado que se ha inscrito en el sistema y su solicitud es aceptado por un usuario administrador este es Activado y recibirá un correo con una notificación y una contraseña generada
	R3.3: Cuando el usuario ingresa por primera vez se le solicitará la información requerida para completar su registro con los siguientes datos:
	Estos son obligatorios
		•	País 
		•	Contraseña (nueva, debe tener 6-10 debe contener al menos 1 mayúscula, 1 minúscula, 1 número)
		•	Intereses
		•	Fecha de nacimiento
		•	Genero 
	Estos son opcionales
		•	Dirección Residencia
		•	Dirección trabajo
		•	Ocupación actual
		•	Teléfono
		•	Foto 
		•	Indicar si su perfil es público o no (es decir permitir que otros usuarios puedan ver su información o no)
		Si no se tienen los datos obligatorios no le permitirá continuar	
	R3.4: Cada usuario egresado puede ver todos sus datos, pero solo puede editar:
		•	Correo
		•	Contraseña  
		•	Intereses
		•	Foto
		•	Dirección Residencia
		•	Dirección trabajo
		•	Ocupación actual
		•	Teléfono 		
	R3.5: El usuario egresado debe tener una sección círculo de amigos (lista de amigos) y puede agregar y eliminar amigos.
	R3.6: El usuario debe poder desactivar su cuenta, en ese estado el usuario puede ingresar y visualizar eventos, noticias, actividades, perfiles, mas no inscribirse ni agregar ni borrar amigos, no recibirá notificaciones vía email (solo la notificación de que su cuenta ha sido activada).
	R3.7: Cuando un administrador suba una nueva actividad o un nuevo evento se enviará un correo con la notificación del evento o actividad a los usuarios egresados que tengan esos mismos intereses de lo subido.
	R3.8: Los usuarios egresados podrán suscribirse a las noticias, eventos, actividades para así obtener notificación vía email en cualquier cambio de la noticia o evento (de cada cambio).	
	R3.9: El sistema le sugerirá amigos que contengan datos similares como interés, de una lista que siempre estará visible en el área de noticias (única para cada usuario) esta lista se le actualizará tras cada login.

Inicio de sesión/registro:
	R4.1: Todo usuario debe ingresar al sistema con Correo electrónico y contraseña, además debe indicar el tipo de usuario sea Súper Usuario, Administrador y usuario egresado
	R4.2: Cuando un usuario realiza el logín debe enviarlo a una vista según su usuario sea:
		•	Súper usuario: debe redirigirlo a la lista de administradores existentes y la lista de solicitudes de activación de administrador
		•	Usuario administrador: le debe de llevar a la lista de sus actividades, y debe tener donde acceder a su perfil
		•	Usuario egresado: Debe de llevarle al menú de opciones de usuario, el cual contiene acceso a módulo de noticias, perfil, círculo de amigos.
	R4.3: En la vista de login debe estar la opción de recuperar contraseña mediante el correo electrónico (se recupera según el tipo de cuenta indicado)
	R4.4: Debe haber un Captcha antes del login, solicitud de inscripción y ¿olvidaste contraseña?
	R4.5: Al inscribirse el usuario el email debe ser validado (formato correcto) y debe ser único para cada DNI es decir puede existir un Administrador con un email y puede existir un usuario egresado con su mismo DNI podrá tener el mismo Email o otro; pero no podrán existir 2 usuarios egresados (cada uno con un DNI diferente) con un mismo email.
			
Aplicativo - web
	R5.1: Todo usuario en el sistema debe poder visualizar listas de actividades, noticias
