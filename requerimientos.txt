Requerimientos - Funcionales
	Super usuario:
	R1.1: El sistema requiere tener un super usuario unico, debe tener Nombre de usuario y conteaseña; este debe de tener permisos para Crear, consultar, Modificar y eliminar perfiles de administrador.
	
	R1.2: al momento de crear un usuario administrador este posee los datos de DNI, Nombre completo, Dirección, Teléfono, Ciudad, correo, una contraseña, pero minimamente debe contener los datos de DNI, Nombre, Correo y contraseña.
	
	R1.3: Cuando un usuario administrador solicite inscribirse el sistema debe confirmar los datos consultando en la base de datos la información subida por el usuario y se le notificará al super usuario
	
	R1.4: El super usuario podrá visualizar la lista de solicitud donde podrá aceptar o rechazar de las resultantes por el requerimiento 1.3
	
	Administradores:
	R2.1: Los usuarios administradores tienen permisos para crear, modificar, visualizar, eliminar perfiles de egresados, desactivar usuarios de egresados
	
	R2.2: los datos requeridos para la creación de un usuario egresado son Nombre, Pais, DNI, Correo, Contraseña, Intereses, Edad, Genero y confirmar para el manejo de datos sencibles. Minimanente debe contener los datos de nombre, correo, contraseña y pais.
	
	R2.3: El usuario administrador posee la facultad de agregar, modificar contenido de Audio, Imagenes, Videos, o otros archivos en el modulo de noticias.
	
	R2.4: El contenido multimedia subido por el administrador se mostrarán en una sección de noticiaso actividades para los usuarios egresados
	
	R2.5: El administrador puede modificar sus propios datos y visualizarlos.
	
	R2.6: Cuando un usuario egresado solicite inscribirse el sistema debe confirmar los datos consultando en la base de datos la información subida por el usuario y se le notificará al administrador
	
	R2.7: El administrador podrá visualizar la lista de solicitud donde podrá aceptar o rechazar de las resultantes por el requerimiento 2.6
	
	R2.8: un usuario puede inscribirse para ser administrador del sistema de la forma como lo hace el usuario egresado como lo indica el requerimiento 3.1, 3.2 con la diferencia que quien lo Acepta es el Super usuario.
	
	Usuarios Egresados:
	R3.1: Para inscribirse en el sistema el usuario egresado primero dbee hacer una solicitud con los siguientes datos Nombres, Apellidos, Programa (donde egreso), Año de egreso, DNI, y correo. Esto es una solicitud que puede ser vista y tratada por los administradores pendiente a ser activada.
	
	R3.2: Una vez que un usuario Egresado que se ha inscrito en el sistema y este es Activado por el administrador recibirá un correo con una notificación y una contraseña generada
	
	R3.3: Cuando el usuario ingresa por primera vez se le solicitará la información requerida para completar us registro con los datos del requerimiento 2.2 con la misma restricción y una nueva contraseña
	
	R3.4: al registrarse el usuario el email debe ser validado (formato correcto) y debe ser unico para cada DNI es decir puede existir un Administrador con un email y puede existir un usuario con su mismo DNI podrá tener el mismo Email.
	
	
	
	R3.5: Cada usuario egresado puede editar, ver sus datos de usuario
	
	R3.6: El usuario egresado debe tener una sección circulo de amigos.
	
	R3.7: El usuario debe poder desactivar su cuenta para con ello impedir su acceso al sistema.
	
	R3.8: Cuando un administrador suba una nueva actividad o un nuevo evento se enviará un correo con la notificación del evento o actividad a los usuarios egresados que tengan esos mismos intereses de lo subido.
	
	R3.9: Los usuarios egresados podrán suscribirse a las noticias y eventos para asi obtener notificación via email en cualquier cambio de la noticia o evento.
	
	Inicio de sesión:
	R4.1: Todo usuario debe ingresar al sistema con Correo electronico y contraseña
	
	R4.2: El usuario debe estar logueado para poder modificar sus datos
	
	R4.3: Cuando un usuario administrador inicia sesión se debe redirigir a la lista de sus actividades (menu principal)
	
	R4.4: Cuando un usuario egresado inicia sesión debe redirigirlo a menu de opciones de usuario, el cual contiene acceso a modulo de noticias, editar perfil, circulo de amigos
	
	R4.5: En la vista de login debe estar la opción de recuperar contraseña meidante el correo electronico
	
	Aplicativo - web
	R5.1: Todo usuario en el sistema debe poder visualizar listas de activiades, noticias
	
	R5.2: Los usuarios administrador deben poder agregar, editar y eliminar las listas de actividades propias
